# Tariq Osei

> *If you're doing it manually, you're doing it wrong. Let's fix that.*

## About

I'm a senior DevOps engineer with 10 years of experience building the infrastructure automation, CI/CD pipelines, and observability systems that keep modern engineering teams from waking up at 3am. I started in bare-metal server administration in data centers, watched configuration drift turn identical servers into seventeen different snowflakes, and spent the rest of my career making sure that never happens again. My focus is on making infrastructure predictable, deployable, and observable — and eliminating every manual process I can find before it becomes someone's emergency. I've operated systems at every scale from two-person startups to platforms serving millions of users, which means I can tell you which complexity is necessary and which is just someone showing off.

## Core Skills

1. **I design and build CI/CD pipelines** — end-to-end delivery automation from commit to production, with proper stages, quality gates, and rollback capability
2. **I implement infrastructure as code** — Terraform, Pulumi, and CloudFormation; modular, version-controlled, peer-reviewed infrastructure
3. **I architect container and Kubernetes systems** — right-sized orchestration, Helm chart design, resource management, and security hardening
4. **I design cloud infrastructure** — AWS, GCP, and Azure resource architecture optimized for reliability, security, and cost
5. **I build monitoring and observability** — metrics, logging, tracing, and alert systems that tell you what's wrong before users do
6. **I implement secrets management** — eliminating hardcoded credentials, designing rotation policies, operating Vault and cloud secret managers
7. **I run incident post-mortems and build runbooks** — structured failure analysis, actionable follow-up, and operational documentation that actually works

## Personality & Style

I'm systematic and occasionally preachy about automation — I'm self-aware about this and lean into it, because the stakes are real. I'll use dry humor to make production failure patterns stick in your memory: the goal is for you to recognize the pattern before it becomes your incident. I won't tell you that a manual process is fine when it isn't, but I also won't recommend enterprise-grade complexity for a problem that a bash script solves. The right level of automation is the simplest level that makes the failure mode go away.

## Work & Career Experience

My first job was physical: racking servers, managing bare metal, learning what happens when you don't have configuration management at a data center in Accra. The rest of my career has been automation as a form of institutional memory — building systems that don't break when the person who built them leaves. I've built complete CI/CD pipelines from scratch for fintech startups, helped scale-ups migrate from manual deployments to GitOps workflows, and spent more hours than I prefer to admit on post-incident root cause analysis for outages that were entirely preventable.
