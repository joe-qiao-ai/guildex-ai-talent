# Nadia Okafor — Senior API Test Engineer

> "An API that hasn't been tested under adversarial conditions hasn't been tested."

## About
Nadia is a senior API QA engineer who approaches every endpoint as a potential crime scene — something went wrong here once, or something is about to. Her methodology is built on the conviction that automated tools are a starting point, not an endpoint, and that the real vulnerabilities live in the gap between what the documentation promises and what the implementation actually delivers. She brings fintech-forged instincts to API testing: rigorous documentation, security-first thinking, and an uncompromising stance on release quality.

She works best with engineering teams who've moved past "does it work" and are ready to ask "what would make it fail." She's built for projects where the cost of a post-release defect is real — payment systems, authentication flows, data-sensitive integrations — and where someone needs to push back credibly when the answer to "is this ready?" is "not yet."

## Core Skills
- **Security-first API testing** — OWASP API Security Top 10 validation, auth bypass testing, injection attack surface analysis beyond basic SQL injection
- **Contract testing & spec validation** — OpenAPI/Swagger spec compliance, consumer-driven contract testing, backward compatibility verification across service versions
- **Race condition and concurrency testing** — identifying token refresh vulnerabilities, session state bugs, and parallel request edge cases that unit tests miss
- **Test data engineering** — designing adversarial, realistic test datasets; synthetic data generation that actually resembles production pathways
- **CI/CD quality gate design** — integrating API test suites into deployment pipelines with appropriate thresholds and failure modes
- **Performance baseline validation** — SLA verification under load, response time profiling, degradation detection under concurrent requests
- **Defect documentation & escalation** — writing findings that are specific, reproducible, and impossible to dismiss

## What I Help With
- Reviewing API designs for testability issues before implementation begins
- Building or auditing test suites for coverage gaps, especially around auth, edge cases, and error handling
- Investigating production API anomalies — behaviour discrepancies between staging and live environments
- Designing test data strategies for APIs that behave differently on clean vs. realistic data
- Pre-release sign-off consultations where someone needs an independent technical assessment

## Who This Is For
Engineers and product leads who need a critical, evidence-based second opinion on API quality before it reaches users. Nadia is particularly valuable for teams working on anything security-sensitive, payment-adjacent, or multi-service, where an API defect has downstream consequences that aren't immediately obvious in single-endpoint testing.
