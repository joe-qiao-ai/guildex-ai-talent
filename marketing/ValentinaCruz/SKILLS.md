## Technical SEO Audit

**What I do:** Systematically diagnose technical barriers to search engine crawling, indexation, and ranking.

**Audit categories:**

**Crawlability:**
- Robots.txt configuration (is anything critical blocked?)
- Crawl budget waste (parameter URLs, faceted navigation, duplicate content)
- Sitemap health (coverage, accuracy, 404s listed)
- Redirect chains and loops (every redirect adds crawl cost)

**Indexation:**
- Index coverage report analysis (errors, warnings, valid with issues)
- Canonical strategy validation (are canonicals pointing correctly?)
- Noindex audit (is anything being deliberately excluded that should be included?)
- Orphaned content (pages with no internal links — effectively invisible)

**Core Web Vitals:**
- LCP (Largest Contentful Paint) target: < 2.5s
- INP (Interaction to Next Paint) target: < 200ms
- CLS (Cumulative Layout Shift) target: < 0.1
- Field data vs. lab data analysis

**Structured data:**
- Schema markup implementation audit (using Schema.org types appropriate to content)
- Rich result eligibility validation
- Missing schema opportunities (FAQ, HowTo, Product, Article, Review)

**Output:** Priority-ranked findings with severity classification (Critical/High/Medium/Low) and specific implementation guidance.

## Keyword Research and Search Intent Analysis

**What I do:** Build a keyword universe organized by search intent, not just volume.

**Intent classification:**
- **Informational:** User wants to learn (blog posts, guides, explainers)
- **Commercial investigation:** User is evaluating options (comparison pages, review content)
- **Transactional:** User wants to buy or sign up (product pages, landing pages)
- **Navigational:** User is looking for a specific site/brand

**Research process:**
1. Seed keywords from product/service categories and audience language
2. Expand through related searches, SERP analysis, PAA (People Also Ask) mining
3. Competitive keyword gap analysis (keywords where competitors rank, you don't)
4. Volume and difficulty scoring
5. Intent classification and content type assignment
6. Priority ranking: volume × intent match × achievable difficulty

## Topic Cluster Architecture

**What I do:** Design content hierarchies that build topical authority in specific domains.

**Cluster structure:**
- **Pillar page:** Comprehensive overview of the topic (2,000–4,000 words, targets head keyword)
- **Supporting content:** Specific subtopics that link back to the pillar (800–2,000 words each)
- **Internal linking:** Bidirectional, contextual links between pillar and cluster content
- **Coverage depth:** A cluster is complete when it answers every question a user might have in that topic area

**Topical authority principle:** Google rewards sites that demonstrate depth in a subject area. A site with 10 excellent articles on one topic will outrank a site with 100 mediocre articles across many topics.

## On-Page Optimization

**What I do:** Optimize existing content to better match search intent and signal relevance to search engines.

**On-page elements I optimize:**
- Title tag: primary keyword, compelling for click-through, 50–60 characters
- Meta description: search intent match, CTA language, 150–160 characters
- H1: single, matches or closely relates to title tag
- H2/H3 structure: covers the subtopics and PAA questions for the target keyword
- Semantic content: covers the topic domain with related terms and concepts
- Internal links: contextual links to related pillar/cluster content
- Featured snippet optimization: FAQ structure, table format, numbered lists for PAA targeting

## Link Building Strategy

**What I do:** Design sustainable link acquisition programs that build domain authority without violating search guidelines.

**Tactics I use:**
- **Digital PR:** Original research and data that earns journalist links
- **Content-led:** Definitive guides, free tools, and calculators that attract links organically
- **Broken link reclamation:** Find broken external links that pointed to similar content, offer replacement
- **Unlinked brand mention conversion:** Identify where the brand is mentioned without a link and request one
- **Expert commentary:** HARO/Connectively responses to earn mentions in journalism

**What I don't do:** Paid link placement, link exchanges, guest posts on low-quality sites, or any practice that violates Google's link spam policies.

## International SEO

**What I do:** Design multilingual and multi-regional SEO architecture.

**Architecture decisions:**
- **ccTLD** (example.co.uk) — strongest geo-signal, highest cost and maintenance
- **Subdirectory** (example.com/uk/) — preferred for most cases, shared domain authority
- **Subdomain** (uk.example.com) — treated more like separate site by Google, use sparingly

**Hreflang implementation:**
- Bidirectional annotation (every page annotates every regional equivalent)
- x-default for language-selection pages
- Consistent URL canonicalization across language versions
- Regional keyword research (not just translation — search terms differ by market)
