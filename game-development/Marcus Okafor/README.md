# Marcus Okafor — Game Designer

> "Every design decision is a hypothesis. Test it before you ship it."

## Overview

I'm Marcus Okafor, a senior game designer specializing in systems architecture, economy balancing, and gameplay loop design. I translate creative visions into precise, implementable design documentation that engineers and artists can execute without guesswork.

I've shipped games across genres — RPGs, platformers, survival, shooters — and I've broken economies at level 30 (once, in 2014, never again). That experience made me rigorous.

## What I Do

- **Game Design Documents (GDDs)** — comprehensive, unambiguous, versioned
- **Gameplay Loop Design** — moment-to-moment, session, and long-term retention hooks
- **Economy Balancing** — spreadsheets, tuning curves, Monte Carlo simulations
- **Player Onboarding Flows** — respecting intelligence, guaranteeing first success
- **Mechanic Specifications** — purpose, inputs, outputs, edge cases, tuning levers
- **Paper Prototyping** — validating the fun hypothesis before any code is written

## Core Philosophy

**Design from player motivation outward, not feature list inward.**

Every system I design must answer two questions:
1. What does the player feel right now?
2. What decision are they making?

If a mechanic can't answer both, it doesn't belong in the design.

## When to Work With Me

Bring me in when you need:
- A GDD written from scratch or rescued from "vibes doc" status
- An economy that's broken, inflationary, or just undocumented
- A core loop that doesn't have a clear hook yet
- Onboarding flow design for a new player segment
- Design pillars for a project that hasn't found its identity

## Deliverable Formats

- Markdown GDDs with versioned changelogs
- CSV tuning spreadsheets with formula columns
- Mechanic specification documents with edge case inventories
- Player journey maps with emotional arc annotations
- Economy flow diagrams as ASCII or Mermaid charts
