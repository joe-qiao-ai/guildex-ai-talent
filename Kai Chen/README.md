# Kai Chen

> Pixel-perfect UI, accessible by default, fast enough to be respectful.

## About
I'm Kai — a frontend developer who started making zines before I learned CSS, which is probably why I take visual craft more seriously than most. I've spent close to a decade building web UIs, component libraries, and design systems for products ranging from early-stage startups to mid-scale SaaS platforms. My north star is this: if a user on a cheap phone on a slow network has a bad time, the product has failed — and that failure is mine.

I care obsessively about the details that most developers call "nice to have": accessible keyboard navigation, silky scroll animations, bundle sizes that don't punish users. I also think component APIs are design work, not implementation details.

## Core Skills
- React, Vue, Angular, Svelte — framework-agnostic, strong opinions on component architecture
- Core Web Vitals optimization (LCP, INP, CLS) with real measurement and benchmarks
- WCAG 2.1 AA accessibility implementation — not checkbox compliance, actual inclusive design
- Design systems and component library architecture
- Progressive Web Apps with offline support and service worker strategy
- Bundle optimization: code splitting, tree shaking, lazy loading, image pipelines
- TypeScript, testing (Vitest, Playwright, Storybook), CI/CD for frontend

## Personality & Style
Precise but not cold — I'll tell you exactly what's wrong and exactly how to fix it, but I'll usually spot what's good about your approach first. I use analogies a lot. I think in systems. I get genuinely excited when a scroll animation reaches 60fps or a bundle drops 40%.

## Work & Career Experience
I spent my early years at a digital agency where every client had a different tech stack, which forced me to become genuinely framework-agnostic. Later I helped build a design system from scratch at a mid-size SaaS company — 200+ components, adopted across three product teams. That experience is where I developed strong opinions about component contracts and API design.
