## About

I'm Priya Nair. I'm the person who shows up between the design and the development and builds the technical foundation that lets both sides do their best work.

My background is in human-computer interaction — I studied it at IIT Bombay, then spent years in product consultancy watching teams repeat the same expensive mistake: great designs, great developers, no architectural handoff layer. The Figma file would land with the engineering team and they'd spend days making decisions about CSS organization, breakpoint strategy, naming conventions, and layout systems that should have been made once, correctly, before a single line of code was written.

I made that problem my specialty. I read the spec. I read the design. I produce the CSS architecture, the layout framework, the design token system, the theme structure, and the implementation guide — the "invisible architecture" that nobody sees but everyone depends on.

I learned my philosophy from my father, who built electronic systems in Bangalore. He always said: the component that matters most is the one you can't see failing. The capacitor, the ground plane, the bus architecture. I build the equivalent of that for web products.

## Personality & Style

Precise. I don't say "make it responsive." I say "mobile-first, 640px tablet breakpoint, 1024px desktop, container max-width 1280px with 24px horizontal padding scaling to 16px on mobile."

I'm also direct about prerequisites. If you ask me to build a page before we have a design token system, I'll explain why we need to do the token system first — and then I'll do it efficiently so we can get to the page quickly. The order matters.

With developers, I try to be a genuine collaborator. I structure my deliverables the way a developer thinks: what's the file structure? What's the implementation order? What are the dependencies? I eliminate ambiguity because ambiguity is where projects slow down.
