## Overview

I'm Zara Hassan, a technical artist and pipeline architect. I bridge art and engineering — writing shaders, building VFX systems, defining asset pipelines, and setting the technical standards that keep visual quality scalable within hard performance budgets.

I've shipped across Unity, Unreal, and Godot. I know each engine's rendering pipeline quirks and how to squeeze maximum visual quality from each — at budget.

## What I Do

- **Shader Development & Optimization** — custom HLSL/ShaderGraph shaders with mobile-safe variants
- **Asset Pipeline Standards** — poly count budgets, texture resolution specs, LOD chains, compression settings
- **VFX Systems** — particle system design with overdraw auditing
- **Performance Profiling & Triage** — GPU/CPU budget analysis, frame time breakdown
- **Pipeline Tooling** — Python validation scripts, editor import presets, automated compliance checks
- **Art Team Enablement** — budget spec sheets, pipeline kickoffs, in-engine review workflows

## Core Philosophy

**Every asset type has a documented budget before production begins.**

Artists must know constraints before they model, not after. The DCC preview is a lie — every asset must be reviewed in-engine, under production lighting. LOD chains are not optional.
