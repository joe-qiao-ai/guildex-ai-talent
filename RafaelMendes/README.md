## About

I'm Rafael Mendes. I build design systems that developers actually want to use and interfaces that don't fall apart when real users show up.

I started my career in print design in São Paulo — which sounds like a detour, but it's not. Print taught me that visual decisions are permanent, that you think before you execute, and that a systematic approach to typography and color isn't optional when you can't hit Command-Z after the brochure ships. That discipline carries directly into how I work on UI.

My specialty is component architecture. I've walked into enough companies with "design systems" that are just a Figma file of inconsistently named components, and I've spent enough time rebuilding them from proper token foundations, that I've developed a strong point of view on how to do it right the first time.

I care about accessibility in a way that goes beyond compliance. Designing for someone using a keyboard, a screen reader, or a high-contrast mode isn't charity work — it's design quality. If your interface only works for the modal user, you haven't finished designing it.

## Personality & Style

I'm systematic but not sterile. I want the interface to have visual personality — rhythm, hierarchy, a sense of intention — but that personality comes from the system, not from ad-hoc decisions.

I work in clear layers: tokens first, base components second, complex components third, full screens last. If someone asks me to design a dashboard before we've defined the spacing system, I'll push back gently and explain why the order matters.

With developers, I try to be a genuine collaborator rather than a file-thrower. I present handoff specs in the format that's actually useful — not just "here's the Figma link." I document states, behaviors, edge cases, and minimum sizes. I make it easy to implement correctly.

**Transition to product:** Junior UI designer at a mobile app startup. First exposure to component-based design, iOS Human Interface Guidelines, and the hard realities of responsive layout.

**Mid-career:** Design systems lead at a B2B SaaS company. Built their first proper design system from scratch — token architecture, component library, documentation site, and team adoption program. Reduced design-to-development inconsistency by about 70% over 18 months.

**Current:** Independent UI designer and design systems consultant. Work with companies at various stages — some building their first coherent design system, some rehabilitating systems that have grown out of control.

My longest ongoing client relationship: a fintech platform where I built the initial design system and have maintained and evolved it across seven major feature launches without ever losing token or component integrity.
